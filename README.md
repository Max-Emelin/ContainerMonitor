# Контейнерный мониторинг

Проект для мониторинга состояния Docker контейнеров, пингования их IP-адресов и отображения данных на веб-странице.

## Функционал

1. **Backend-сервис**:
   - Реализует RESTful API для взаимодействия с базой данных.
   - Обеспечивает добавление новых данных и предоставление текущих данных для фронтенда.

2. **Frontend-сервис**:
   - Разработан на JavaScript с использованием библиотеки React.
   - Отображает таблицу с данными о контейнерах: IP-адрес, время последнего пинга, дата последней успешной попытки.

3. **База данных PostgreSQL**:
   - Хранит информацию о контейнерах и результатах их пинга (IP-адрес, время пинга, дата последнего успешного пинга).

4. **Сервис Pinger**:
   - Получает список всех контейнеров Docker.
   - Пингует контейнеры с заданным интервалом.
   - Отправляет результаты пинга в базу данных через Backend API.


## Шаги запуска

1. **Клонировать репозиторий**:

    ```bash
    git clone https://github.com/Max-Emelin/ContainerMonitor.git
    cd container-monitoring
    ```

2. **Создать `.env` файл** в корне проекта:

    ```env
    SERVER_PORT=8080
    DB_HOST=db
    DB_PORT=5432
    DB_USER=postgres
    DB_PASSWORD=qwerty
    DB_NAME=container_monitor_db
    DB_SSLMode=disable
    PING_INTERVAL_SEC=30
    ```

3. **Запустить через Docker Compose**:

    Убедитесь, что у вас установлены Docker и Docker Compose, затем выполните:

    ```bash
    docker-compose up --build -d
    ```

4. **Доступ к приложению**:
    - **Frontend** (интерфейс): http://localhost:3000
    - **Backend** (API): http://localhost:8080

5. **Остановка контейнеров**:

    ```bash
    docker-compose down
    ```
